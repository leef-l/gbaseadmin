# GBaseAdmin 权限框架架构设计文档

## 背景

构建一套基于 GoFrame v2 MonoRepo（后端）+ Vben Admin（前端）的 RBAC 基础权限框架。
包含：部门管理、角色管理、菜单管理、用户管理，以及一个可扩展的代码生成器（最高优先级）。

**技术选型确认：**
- 后端：GoFrame v2 MonoRepo
- 前端：Vben Admin `apps/web-antd`（Ant Design Vue）
- 认证：JWT（access_token 2h + refresh_token 7d）
- ID 策略：Snowflake，JSON 序列化为 string 防前端精度丢失
- 代码生成器触发方式：CLI 命令行

## 二、数据库设计

> 所有 ID 使用 Snowflake，BIGINT UNSIGNED；前端传输时序列化为 string 防精度丢失。
> 所有表包含公共字段：`created_by`, `dept_id`, `created_at`, `updated_at`, `deleted_at`

### 2.1 部门表 `dept`

```sql
CREATE TABLE dept (
    id         BIGINT UNSIGNED NOT NULL            COMMENT '部门ID',
    parent_id  BIGINT UNSIGNED NOT NULL DEFAULT 0  COMMENT '上级部门ID',
    title      VARCHAR(50) NOT NULL                COMMENT '部门名称',
    username   VARCHAR(50)                         COMMENT '部门负责人',
    email      VARCHAR(100)                        COMMENT '负责人邮箱',
    sort       INT NOT NULL DEFAULT 0              COMMENT '排序',
    status     TINYINT(1) NOT NULL DEFAULT 1       COMMENT '状态:0=关闭,1=开启',
    created_by BIGINT UNSIGNED                     COMMENT '创建人ID',
    dept_id    BIGINT UNSIGNED                     COMMENT '所属部门ID',
    created_at DATETIME                            COMMENT '创建时间',
    updated_at DATETIME                            COMMENT '更新时间',
    deleted_at DATETIME                            COMMENT '删除时间',
    PRIMARY KEY (id)
) COMMENT='部门表';
```

### 2.2 角色表 `role`

```sql
CREATE TABLE role (
    id         BIGINT UNSIGNED NOT NULL            COMMENT '角色ID',
    parent_id  BIGINT UNSIGNED NOT NULL DEFAULT 0  COMMENT '上级角色ID',

    data_scope TINYINT NOT NULL DEFAULT 1          COMMENT '数据范围:1=全部,2=本部门及以下,3=本部门,4=仅本人,5=自定义',
    sort       INT NOT NULL DEFAULT 0              COMMENT '排序',
```

## 三、代码生成器设计（最高优先级）

### 3.1 核心流程

```
数据库表结构 → 解析器(Parser) → 元数据(TableMeta/FieldMeta)
                                        ↓
                        后端代码生成器 → api / controller / logic / service / model / consts
                                        ↓
                        前端代码生成器 → api.ts / list.vue / form.vue / types.ts
```

### 3.2 CLI 使用方式

```bash
# 生成单张表（前后端全量）
go run codegen/main.go --table=dept

# 生成多张表
go run codegen/main.go --table=dept,role,menu,users

# 只生成后端
go run codegen/main.go --table=dept --only=backend

# 只生成前端
go run codegen/main.go --table=dept --only=frontend

# 强制覆盖已存在文件
go run codegen/main.go --table=dept --force
```

### 3.3 字段 → 前端组件映射规则

| 字段名规则 | 前端组件 | 说明 |
|---|---|---|
| `parent_id` | `ApiTreeSelect` 单选 | 从 `/tree` 接口加载 |
| `*_id`（其他外键） | `ApiSelect` 单选下拉 | 需指定关联接口 |
| `parent_ids` | `ApiTreeSelect` 多选 | |
| `*_ids` | `ApiSelect` 多选下拉 | |
| `status` / `is_*` | `Switch` 或 `RadioGroup` | 解析备注枚举值 |
| `*_image` / `*_img` / `avatar` / `*_photo` | `ImageUpload` | |
| `*_file` / `*_attachment` | `FileUpload` | |
| `*_content` / `*_body` / `*_html` | `RichTextEditor` | |
| `*_json` / `*_config` / `*_settings` | `JsonEditor` | |
| `*_password` / `*_pwd` | `PasswordInput` | |
| `*_url` / `*_link` | `Input` + URL 验证 | |
| `*_at`（时间字段） | `DateTimePicker` | |
| `sort` / `order` / `*_num` | `InputNumber` | |
| `type` / `level` / `grade` | `Select`（枚举） | 解析备注枚举 |
| `icon` | `IconPicker` | |
| 含枚举备注（`x=y,...`） | `Select` / `RadioGroup` | 自动生成 consts |
| 其他 VARCHAR | `Input` | |
| TEXT / LONGTEXT | `Textarea` | |

### 3.4 Label & Consts 生成规则

字段备注格式：`状态:0=关闭,1=开启`
- `:` 前部分 → `label = "状态"`
- `:` 后部分 → 生成 Go consts + 通过接口暴露给前端

字段备注无 `:` → 直接用备注作为 label

**后端 consts 示例**（`internal/consts/` 目录）：
```go
const (
    StatusClose = 0 // 关闭
    StatusOpen  = 1 // 开启
)
```

提供 `GET /api/system/v1/consts` 接口，返回所有枚举常量供前端统一使用。

### 3.5 有 `parent_id` 字段的特殊规则

后端自动生成两个列表接口：
- `GET /list` → 普通分页列表
- `GET /tree` → 树形结构（递归组装 `children`）

### 3.6 可扩展性设计

- 模板引擎：Go `text/template`，模板文件独立于生成器逻辑，可自由替换
- 元数据结构（`FieldMeta`, `TableMeta`）统一定义，新增组件类型步骤：
  1. 在 `parser/` 中增加字段识别规则
  2. 在 `templates/frontend/` 中增加对应模板片段
- `codegen.yaml` 配置文件支持覆盖默认规则（如自定义字段→组件映射）

## 四、后端核心设计

### 4.1 认证：JWT

- 登录返回 `access_token`（2h）+ `refresh_token`（7d）
- Header：`Authorization: Bearer <token>`
- GoFrame 中间件解析 JWT Claims，获取 `userId` / `deptId` / `roles`

### 4.2 Snowflake ID 精度处理

```go
// utility/snowflake/snowflake.go
// ID 字段 JSON 序列化输出为 string，防 JS 精度丢失
type JsonInt64 int64

func (i JsonInt64) MarshalJSON() ([]byte, error) {
    return []byte(`"` + strconv.FormatInt(int64(i), 10) + `"`), nil
}

func (i *JsonInt64) UnmarshalJSON(b []byte) error {
    s := strings.Trim(string(b), `"`)
    n, err := strconv.ParseInt(s, 10, 64)
    *i = JsonInt64(n)
    return err
}
```

API 响应结构体中所有 ID 字段统一使用 `JsonInt64` 类型。

### 4.3 统一响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": { ... }
}
```

### 4.4 代码分层规范

```
Controller  → 参数绑定、基础校验、调用 service、返回响应（不含业务逻辑）
Service     → 接口定义（由 gf gen service 从 logic 自动生成）
Logic       → 具体业务实现，调用 DAO
DAO         → 数据库操作（由 gf gen dao 从数据库表自动生成）
```

## 五、前端核心设计（Vben Admin - web-antd）

### 5.1 Snowflake ID 处理

后端统一返回 string 类型 ID，前端请求参数也以 string 传递，无需 BigInt。

### 5.2 每个模块生成文件

```
apps/web-antd/src/
├── api/system/{module}.ts              # API 调用封装
├── views/system/{module}/
│   ├── index.vue                       # 列表页（含树形/普通切换）
│   └── modules/
│       └── form.vue                    # 新建/编辑表单弹窗
└── router/routes/modules/system.ts    # 路由注册
```

### 5.3 角色授权特殊交互

- **资源权限**：弹出菜单树，CheckBox 多选
- **数据权限**：
  - 下拉选择数据范围（1=全部 / 2=本部门及以下 / 3=本部门 / 4=仅本人 / 5=自定义）
  - 当选择 `5=自定义` 时，显示部门树 CheckBox 多选

### 5.4 用户表单特殊交互

- **部门字段**：树形下拉多选，仅显示当前用户有权限的部门及其下级
- **角色字段**：树形下拉多选，仅显示当前用户有权限的角色及其下级

## 六、实施顺序

```
Phase 1：初始化项目
  1. gf init gbaseadmin -m       → MonoRepo 根
  2. gf init app/system -a       → 权限服务
  3. 创建数据库表（共 8 张）

Phase 2：代码生成器（最高优先级）
  4. 开发 codegen 工具
     - parser：读取 DB 表结构 → TableMeta/FieldMeta
     - generator/backend：生成 api/controller/logic/service/model/consts
     - generator/frontend：生成 api.ts/list.vue/form.vue/types.ts
  5. 以 dept 表跑通全流程验证

Phase 3：生成 CRUD 代码
  6. codegen 生成 dept/role/menu/users 前后端代码
  7. 手动补充特殊业务逻辑：
     - 树形接口（递归/迭代组装）
     - 角色授权（grant/menu, grant/dept）
     - 数据权限过滤（data_scope）

Phase 4：权限框架联调
  8. JWT 登录认证
  9. 菜单动态路由（后端模式）
  10. 按钮级权限（permission 字段）
  11. 数据权限过滤中间件
```
