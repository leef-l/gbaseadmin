## 二、前后端功能差异

以下功能在前端和后端之间存在**不一致**或**对不上**的情况。

### 2.1 后端有但前端未使用的接口

| 后端接口 | 路径 | 说明 |
|----------|------|------|
| 余额支付（管理端） | `POST /play/payment/pay_by_balance` | 后端 `payment_enhance.go` 已实现，前端无 enhance.ts、无调用入口 |
| 独立搜索接口 | `POST /playapi/search` | WAP 搜索页用 `getCoachList + getGoodsList` 代替，未调用独立搜索 API |
| 单条消息已读 | `POST /playapi/message/read` | WAP 消息页只用了 `markAllRead`，`markRead` 从未调用 |

### 2.2 前端传参与后端/类型定义不匹配

| 前端文件 | 问题 |
|----------|------|
| `goods/detail.tsx` | `getReviewList({ goodsId })` 但 API 类型定义只有 `coachId` 参数 |
| `mine/profile.tsx` | `updateMember({ birthday })` 但 API 类型定义无 `birthday` 字段 |
| `order/confirm.tsx` | `createOrder({ coachId })` 但 API 类型定义无 `coachId` 字段 |
| `mine/index.tsx` | 使用 `userInfo.levelTitle` 但 store 类型只定义了 `levelName` |

### 2.3 路由版本选择错误

管理端部分模块同时有 `index.vue` 和 `enhance.vue` 两个版本，路由加载的是功能较弱的 `index.vue`：

| 模块 | 路由加载 | 应使用 | 差异 |
|------|----------|--------|------|
| 活动管理 | `activity/index.vue` | `enhance.vue` | index 对所有活动类型显示"管理步骤"按钮（错误），enhance 仅对 type=4 显示（正确） |
| 订单管理 | `order/index.vue` | `enhance.vue` | index 金额显示分/显示原始ID，enhance 金额÷100显示元/显示关联名称 |
| 利润流水 | `profit_log/index.vue` | `enhance.vue` | 同上 |
