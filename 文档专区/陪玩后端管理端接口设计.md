# 陪玩平台后端管理端接口设计

> 基于 GBaseAdmin 框架（GoFrame v2 MonoRepo），参考 system 模块代码风格
> 所有 ID 使用 Snowflake（uint64），JSON 序列化为 string（`snowflake.JsonInt64`）
> 路由前缀：`/api/play`，应用：`admin-go/app/play`

---

## 一、基础配置模块

### 1.1 会员等级管理

#### 1.1.1 创建会员等级

- **路径**：`POST /member_level/create`
- **说明**：创建会员等级配置

**请求参数**：

```go
type MemberLevelCreateReq struct {
    g.Meta   `path:"/member_level/create" method:"post" tags:"会员等级" summary:"创建会员等级"`
    Title    string `json:"title" v:"required#等级名称不能为空" dc:"等级名称"`
    Level    int    `json:"level" v:"required|in:1,2,3,4,5#等级值不能为空|等级值必须在1-5之间" dc:"等级"`
    Icon     string `json:"icon" dc:"等级图标"`
    MinExp   int    `json:"minExp" v:"required|min:0#最低经验值不能为空|最低经验值不能为负数" dc:"所需最低经验值"`
    Discount int    `json:"discount" v:"required|between:1,100#折扣不能为空|折扣须在1-100之间" dc:"折扣百分比"`
    Sort     int    `json:"sort" dc:"排序（升序）"`
    Status   int    `json:"status" dc:"状态:0=关闭,1=开启"`
}
```

**响应参数**：

```go
type MemberLevelCreateRes struct {
    g.Meta `mime:"application/json"`
}
```

**业务逻辑**：

```
1. 生成 Snowflake ID
2. 校验 level 值是否已存在（同一等级值不能重复）
   - SELECT COUNT(*) FROM play_member_level WHERE level=? AND deleted_at IS NULL
   - 若存在返回错误"该等级值已存在"
3. 插入 play_member_level 记录
4. 返回成功
```

    Orders int    `json:"orders"` // 订单数

```

   FROM play_order
   WHERE order_status=3 AND finish_at >= DATE_SUB(CURDATE(), INTERVAL ? DAY)
   GROUP BY DATE(finish_at)
   ORDER BY date ASC
```
