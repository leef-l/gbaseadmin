# PlayBuddy 宝塔 Nginx + Docker 部署教程

> 域名：`pw.easytestdev.online`
> 架构：宝塔 Nginx 做反向代理 + Docker 跑 MySQL/Redis/Go后端
> 前端：本地构建后由 Nginx 直接托管静态文件

---

## 架构总览

```
                    ┌─────────────────────────────────┐
                    │        宝塔 Nginx (:80/:443)     │
                    │      pw.easytestdev.online       │
                    ├─────────────────────────────────┤
                    │                                   │
                    │  /api/system/*  → 127.0.0.1:8000 │ ← Docker: system
                    │  /api/play/*    → 127.0.0.1:8001 │ ← Docker: play
                    │  /api/playapi/* → 127.0.0.1:8001 │ ← Docker: play
                    │  /wap/*         → 静态文件         │ ← wap-ui 构建产物
                    │  /*             → 静态文件         │ ← vue-vben-admin 构建产物
                    │                                   │
                    └─────────────────────────────────┘
                                    │
                    ┌───────────────┼───────────────┐
                    │         Docker Network          │
                    │                                 │
                    │  mysql:3306    redis:6379       │
                    │  system:8000   play:8001        │
                    └─────────────────────────────────┘
```

## 第一步：服务器准备

### 1.1 安装宝塔面板

```bash
# CentOS
yum install -y wget && wget -O install.sh https://download.bt.cn/install/install_6.0.sh && sh install.sh ed8484bec

# Ubuntu/Debian
wget -O install.sh https://download.bt.cn/install/install-ubuntu_6.0.sh && sudo bash install.sh ed8484bec
```

### 1.2 宝塔面板安装软件

进入宝塔面板 → 软件商店，安装：
- Nginx（任意版本，推荐 1.24+）
- 不需要安装 MySQL/Redis（Docker 里跑）

### 1.3 安装 Docker

```bash
# 宝塔面板 → 软件商店 → 搜索 Docker → 安装
# 或者手动安装：
curl -fsSL https://get.docker.com | sh
systemctl enable docker && systemctl start docker

# 安装 docker-compose
curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" \
  -o /usr/local/bin/docker-compose
chmod +x /usr/local/bin/docker-compose
```

### 1.4 安装 Node.js（构建前端用）

```bash
# 宝塔面板 → 软件商店 → Node.js版本管理器 → 安装 Node 22.x
# 或手动：
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
apt-get install -y nodejs

# 安装 pnpm
npm install -g pnpm
```

## 第三步：Docker 启动后端服务

### 3.1 创建环境变量文件

```bash
cd /www/wwwroot/project/gbaseadmin/admin-go
cat > .env << 'EOF'
# ─── 数据库 ───
MYSQL_ROOT_PASSWORD=YourRootPass123!
MYSQL_DATABASE=gbaseadmin
MYSQL_USER=gbaseadmin
MYSQL_PASSWORD=YourDBPass123!
MYSQL_PORT=3306

# ─── Redis ───
REDIS_PASSWORD=YourRedisPass123!
REDIS_PORT=6379

# ─── JWT ───
JWT_SECRET=your-jwt-secret-key-change-me

# ─── 端口 ───
SYSTEM_PORT=8000
PLAY_PORT=8001
EOF
```

### 3.2 精简 docker-compose（生产版）

创建生产用的 compose 文件，去掉 frontend/wap/adminer 等开发服务：

```bash
cat > docker-compose.prod.yml << 'YAML'
version: '3.8'

services:
  mysql:
    image: mysql:8.0
    container_name: gbaseadmin_mysql
    restart: unless-stopped
    command: >
      --character-set-server=utf8mb4
      --collation-server=utf8mb4_unicode_ci
      --default-authentication-plugin=caching_sha2_password
      --max-connections=500
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}
      MYSQL_DATABASE: ${MYSQL_DATABASE}
      MYSQL_USER: ${MYSQL_USER}
      MYSQL_PASSWORD: ${MYSQL_PASSWORD}
      TZ: Asia/Shanghai
    volumes:
      - mysql_data:/var/lib/mysql
      - ./deploy/mysql/init.sql:/docker-entrypoint-initdb.d/init.sql:ro
    ports:
      - "127.0.0.1:${MYSQL_PORT:-3306}:3306"
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "127.0.0.1", "-u", "root", "-p${MYSQL_ROOT_PASSWORD}"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 30s
    networks:
      - gbaseadmin_net

  redis:
    image: redis:7-alpine
    container_name: gbaseadmin_redis
    restart: unless-stopped
    command: redis-server --requirepass ${REDIS_PASSWORD} --appendonly yes --maxmemory 256mb --maxmemory-policy allkeys-lru
    volumes:
      - redis_data:/data
    ports:
      - "127.0.0.1:${REDIS_PORT:-6379}:6379"
    healthcheck:
      test: ["CMD", "redis-cli", "-a", "${REDIS_PASSWORD}", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - gbaseadmin_net

  system:
    build:
      context: .
      dockerfile: deploy/Dockerfile.system
    container_name: gbaseadmin_system
    restart: unless-stopped
    depends_on:
      mysql: { condition: service_healthy }
      redis: { condition: service_healthy }
    environment:
      - GF_GCFG_FILE=config.yaml
      - DB_HOST=mysql
      - DB_PORT=3306
      - DB_USER=${MYSQL_USER}
      - DB_PASS=${MYSQL_PASSWORD}
      - DB_NAME=${MYSQL_DATABASE}
      - REDIS_ADDR=redis:6379
      - REDIS_PASS=${REDIS_PASSWORD}
      - JWT_SECRET=${JWT_SECRET}
      - GF_SERVER_ADDRESS=:8000
      - TZ=Asia/Shanghai
    ports:
      - "127.0.0.1:8000:8000"
    networks:
      - gbaseadmin_net

  play:
    build:
      context: .
      dockerfile: deploy/Dockerfile.play
    container_name: gbaseadmin_play
    restart: unless-stopped
    depends_on:
      mysql: { condition: service_healthy }
      redis: { condition: service_healthy }
    environment:
      - GF_GCFG_FILE=config.yaml
      - DB_HOST=mysql
      - DB_PORT=3306
      - DB_USER=${MYSQL_USER}
      - DB_PASS=${MYSQL_PASSWORD}
      - DB_NAME=${MYSQL_DATABASE}
      - REDIS_ADDR=redis:6379
      - REDIS_PASS=${REDIS_PASSWORD}
      - JWT_SECRET=${JWT_SECRET}
      - GF_SERVER_ADDRESS=:8001
      - TZ=Asia/Shanghai
    ports:
      - "127.0.0.1:8001:8001"
    networks:
      - gbaseadmin_net

volumes:
  mysql_data:
  redis_data:

networks:
  gbaseadmin_net:
    driver: bridge
YAML
```

### 3.3 启动 Docker 服务

```bash
cd /www/wwwroot/project/gbaseadmin/admin-go

# 构建并启动（首次会比较慢，需要下载镜像+编译Go）
docker-compose -f docker-compose.prod.yml up -d --build

# 查看状态
docker-compose -f docker-compose.prod.yml ps

# 查看日志
docker-compose -f docker-compose.prod.yml logs -f system play
```

预期输出：
```
NAME                 STATUS              PORTS
gbaseadmin_mysql     Up (healthy)        127.0.0.1:3306->3306/tcp
gbaseadmin_redis     Up (healthy)        127.0.0.1:6379->6379/tcp
gbaseadmin_system    Up (healthy)        127.0.0.1:8000->8000/tcp
gbaseadmin_play      Up (healthy)        127.0.0.1:8001->8001/tcp
```

## 第四步：构建前端静态文件

### 4.1 构建管理后台（Vben Admin）

```bash
cd /www/wwwroot/project/gbaseadmin/vue-vben-admin

# 安装依赖
pnpm install

# 设置 API 地址（构建时注入）
# 因为 Nginx 和 API 同域，用相对路径即可
cat > apps/web-antd/.env.production << 'EOF'
VITE_GLOB_API_URL=
EOF

# 构建
pnpm build:antd

# 构建产物在：
ls apps/web-antd/dist/
```

### 4.2 构建 WAP 端（Taro H5）

```bash
cd /www/wwwroot/project/gbaseadmin/wap-ui

# 安装依赖
npm install

# 设置 API 地址（生产环境用相对路径，Nginx 代理）
cat > .env.production << 'EOF'
TARO_APP_API=
EOF

# 构建 H5 版本
npm run build:h5

# 构建产物在：
ls dist/
```

### 4.3 创建静态文件目录

```bash
# 管理后台
mkdir -p /www/wwwroot/pw.easytestdev.online/admin
cp -r /www/wwwroot/project/gbaseadmin/vue-vben-admin/apps/web-antd/dist/* \
      /www/wwwroot/pw.easytestdev.online/admin/

# WAP 端
```

## 第五步：宝塔 Nginx 配置

### 5.1 宝塔添加站点

1. 宝塔面板 → 网站 → 添加站点
2. 域名：`pw.easytestdev.online`
3. 根目录：`/www/wwwroot/pw.easytestdev.online/admin`
4. PHP版本：纯静态
5. 点击「提交」

### 5.2 申请 SSL 证书

1. 宝塔面板 → 网站 → `pw.easytestdev.online` → SSL
2. 选择「Let's Encrypt」→ 勾选域名 → 申请
3. 开启「强制 HTTPS」

### 5.3 修改 Nginx 配置

宝塔面板 → 网站 → `pw.easytestdev.online` → 设置 → 配置文件

**删除全部内容**，替换为以下配置：

```nginx
server {
    listen 80;
    listen 443 ssl http2;
    server_name pw.easytestdev.online;

    # ─── SSL（宝塔自动生成的证书路径，按实际修改） ───
    ssl_certificate    /www/server/panel/vhost/cert/pw.easytestdev.online/fullchain.pem;
    ssl_certificate_key /www/server/panel/vhost/cert/pw.easytestdev.online/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384;
    ssl_prefer_server_ciphers on;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;

    # ─── 强制 HTTPS ───
    if ($server_port !~ 443) {
        rewrite ^(/.*)$ https://$host$1 permanent;
    }

    # ─── 基础设置 ───
    client_max_body_size 50m;
    access_log /www/wwwlogs/pw.easytestdev.online.log;
    error_log  /www/wwwlogs/pw.easytestdev.online.error.log;

    # ─── 安全头 ───
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;

    # ─── Gzip ───
    gzip on;
    gzip_min_length 1k;
    gzip_comp_level 6;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript image/svg+xml;
    gzip_vary on;

    # ═══════════════════════════════════════════
    # API 反向代理（Docker 后端）
    # ═══════════════════════════════════════════

    # play 管理端 API
    location /api/play/ {
        proxy_pass http://127.0.0.1:8001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 30s;
        proxy_read_timeout 60s;
    }

    # play C端 API（WAP端调用）
    location /api/playapi/ {
        proxy_pass http://127.0.0.1:8001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 30s;
        proxy_read_timeout 60s;
    }

    # system 管理端 API
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 30s;
        proxy_read_timeout 60s;
    }

    # ═══════════════════════════════════════════
    # WAP 端（H5 静态文件）
    # ═══════════════════════════════════════════

    location /wap/ {
        alias /www/wwwroot/pw.easytestdev.online/wap/;
        index index.html;
        try_files $uri $uri/ /wap/index.html;

        # 静态资源缓存
        location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
            expires 30d;
            add_header Cache-Control "public, immutable";
        }
    }

    # ═══════════════════════════════════════════
    # 管理后台（Vben Admin 静态文件）
    # ═══════════════════════════════════════════

    root /www/wwwroot/pw.easytestdev.online/admin;
    index index.html;

    # Vite 构建的 hashed 资源，长期缓存
    location /assets/ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # index.html 不缓存（确保更新后立即生效）
    location = /index.html {
        add_header Cache-Control "no-cache, no-store, must-revalidate";
        add_header Pragma "no-cache";
        add_header Expires "0";
    }

    # Vue Router history 模式 fallback
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

> **注意**：`/api/play/` 和 `/api/playapi/` 必须放在 `/api/` 前面，Nginx 按最长前缀匹配。

## 常见问题

### Q: Docker 端口被占用
```bash
# 查看端口占用
lsof -i :8000
lsof -i :8001
# 修改 .env 中的端口号
```

### Q: 前端页面空白
- 检查 `dist/` 目录是否有文件
- 检查 Nginx root 路径是否正确
- 检查浏览器控制台是否有 404 错误
- 确认 `index.html` 中的资源路径是否正确

### Q: API 返回 502 Bad Gateway
- 检查 Docker 容器是否正常运行：`docker ps`
- 检查后端日志：`docker logs gbaseadmin_system`
- 确认端口绑定到 `127.0.0.1`

### Q: 数据库连接失败
- 等待 MySQL 完全启动（healthcheck 通过）
- 检查 `.env` 中的密码是否正确
- 进入容器测试：`docker exec -it gbaseadmin_mysql mysql -u root -p`

### Q: WAP 端页面刷新 404
- 确认 Nginx 配置中 `/wap/` 的 `try_files` 指向 `/wap/index.html`
