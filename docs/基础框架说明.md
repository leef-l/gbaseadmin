# GBaseAdmin 基础框架说明

当前仓库已经收缩为一套只保留后台基础能力的框架，范围如下：

- `admin-go/app/system`：登录、用户、角色、部门、菜单、权限
- `admin-go/app/upload`：文件、目录、目录规则、上传配置
- `vue-vben-admin/apps/web-antd/src/`：管理端前端，仅保留 `system` / `upload` 对应模块与公共壳
- `admin-go/codegen/`：代码生成器与初始化 SQL

## 后端应用

| 应用 | 端口 | 路由前缀 | 作用 |
| --- | --- | --- | --- |
| `system` | `8000` | `/api/system` | 权限、组织、账号、菜单 |
| `upload` | `8002` | `/api/upload` | 上传配置、目录、文件记录 |

## 管理端说明

- 路由仍使用 Vben Admin 壳
- 权限模式为 `backend`
- 默认首页为 `/system/dept`
- 动态菜单来自后端 `/system/auth/menus`

## 本地开发

### 后端

```powershell
cd admin-go
go build ./app/system/...
go build ./app/upload/...
```

### 管理端

```powershell
cd vue-vben-admin
pnpm install
pnpm dev:antd
pnpm build:antd
```

### Docker

```powershell
.\docker\dev\compose.ps1 up -d --build
.\docker\dev\compose.ps1 down -v
```

```bash
./docker/dev/compose.sh up -d --build
./docker/dev/compose.sh down -v
```

## 重要约定

- 不要把 `dao/`、`model/do/`、`model/entity/` 当业务层维护
- 菜单删改要同时检查前端入口和 `system_menu` 初始化数据
- 统一响应结构仍为 `{ code, message, data }`
- ID 继续使用雪花 ID / 字符串化 JSON 约定
