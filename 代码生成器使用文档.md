# GBaseAdmin 代码生成器使用文档

## 概述

GBaseAdmin 代码生成器是一个基于数据库表结构自动生成 GoFrame v2 后端代码和 Vben Admin 前端代码的 CLI 工具。

支持特性：
- 自动解析数据库表结构、字段注释、枚举值
- 后端生成 GoFrame v2 分层代码（API/Controller/Logic/Service/Model/Consts/Router）
- 前端生成 Vben Admin 页面（Vxe Table 列表 + Vben Form 表单 + Vben Modal 弹窗）
- 自动识别树形表（含 parent_id）生成树形接口和树形表格
- 自动识别外键关联字段，生成关联查询
- 自动识别枚举注释，生成常量和下拉选项
- 密码字段自动 bcrypt 加密，输出排除
- 软删除支持（deleted_at）
- Snowflake ID + JsonInt64 序列化

## 目录结构

```
admin-go/codegen/
├── main.go                    # CLI 入口
├── config.go                  # 配置加载
├── codegen.yaml               # 默认配置文件
├── bin/codegen                # 编译后的二进制
├── parser/                    # 数据库解析器
│   ├── meta.go                # TableMeta/FieldMeta 结构体
│   ├── parser.go              # 数据库连接和表解析
│   ├── comment_parser.go      # 字段注释解析（提取 label 和枚举值）
│   └── field_mapper.go        # 字段类型映射（Go/TS/组件）
├── generator/
│   ├── backend/generator.go   # 后端代码生成器
│   └── frontend/generator.go  # 前端代码生成器
└── templates/
    ├── backend/               # 后端模板（7 个 .tpl）
    │   ├── api.tpl            # API 请求/响应结构体
    │   ├── controller.tpl     # Controller 层
    │   ├── logic.tpl          # Logic 层业务实现
    │   ├── service.tpl        # Service 接口定义
    │   ├── model.tpl          # DTO 模型
    │   ├── consts.tpl         # 枚举常量
    │   └── router.tpl         # 路由注册
    └── frontend/              # 前端模板（4 个 .tpl）
        ├── types.tpl          # TypeScript 类型定义
        ├── api.tpl            # API 调用函数
        ├── list.tpl           # 列表页（Vxe Table）
        └── form.tpl           # 表单弹窗（Vben Modal + Form）
```

## 快速开始

### 1. 编译

```bash
cd admin-go/codegen
go build -o bin/codegen .
```

### 2. 配置

编辑 `codegen.yaml`：

```yaml
# 数据库连接
database:
  host: 127.0.0.1
  port: 3306
  user: root
  password: "gbaseadmin123"
  dbname: gbaseadmin

# 后端输出目录（相对于 codegen 目录）
backend:
  output: ../app/system/

# 前端输出目录（相对于 codegen 目录）
frontend:
  output: ../../vue-vben-admin/apps/web-antd/src/

# 跳过的公共字段（不生成表单和列表列）
skip_fields:
  - created_at
  - updated_at
  - deleted_at
  - created_by
  - dept_id
```

### 3. 运行

```bash
cd admin-go/codegen

# 生成单表（前后端都生成）
./bin/codegen --table=dept

# 生成多表
./bin/codegen --table=dept,role,menu,users

# 只生成前端
./bin/codegen --table=dept --only=frontend

# 只生成后端
./bin/codegen --table=dept --only=backend

# 强制覆盖已有文件
./bin/codegen --table=dept --force

# 预览模式（只打印文件列表，不实际写入）
./bin/codegen --table=dept --dry-run

# 指定配置文件
./bin/codegen --table=dept --config=./my-config.yaml
```

## 命令行参数

| 参数 | 说明 | 默认值 | 示例 |
|------|------|--------|------|
| `--table` | 要生成的表名，多个用逗号分隔（必填） | - | `--table=dept,role` |
| `--only` | 只生成指定端 | 空（都生成） | `--only=frontend` |
| `--force` | 强制覆盖已存在文件 | false | `--force` |
| `--config` | 配置文件路径 | `./codegen.yaml` | `--config=./dev.yaml` |
| `--dry-run` | 只打印将生成的文件列表 | false | `--dry-run` |

## 生成文件说明

### 后端（每表 7 个文件）

| 文件 | 路径 | 说明 |
|------|------|------|
| API 定义 | `api/system/v1/{table}.go` | Req/Res 结构体 |
| Controller | `internal/controller/{table}/{table}.go` | 控制器，调用 Service |
| Logic | `internal/logic/{table}/{table}.go` | 业务逻辑实现 |
| Service | `internal/service/{table}.go` | 接口定义 |
| Model | `internal/model/{table}.go` | DTO（Input/Output/ListInput/TreeOutput） |
| Consts | `internal/consts/{table}.go` | 枚举常量 |
| Router | `internal/router/{table}.go` | 路由注册 |

### 前端（每表 4 个文件）

| 文件 | 路径 | 说明 |
|------|------|------|
| 类型定义 | `api/system/{table}/types.ts` | TypeScript 接口 |
| API 调用 | `api/system/{table}/index.ts` | 请求函数 |
| 列表页 | `views/system/{table}/index.vue` | Vxe Table + 搜索表单 |
| 表单弹窗 | `views/system/{table}/modules/form.vue` | Vben Modal + Form |

## 字段注释规范

代码生成器通过解析数据库字段注释来生成 label、枚举值和组件类型。

### 基本格式

```sql
`title` VARCHAR(50) COMMENT '部门名称'
-- 生成 label: "部门名称"，组件: Input
```

### 枚举格式

```sql
`status` TINYINT(1) COMMENT '状态:0=关闭,1=开启'
-- 生成 label: "状态"
-- 枚举值: [{Label:"关闭", Value:0}, {Label:"开启", Value:1}]
-- 组件: 2 个选项用 Radio，3+ 个用 Select
-- 同时生成 consts 常量文件
```

### 特殊字段自动识别

| 字段名模式 | 识别为 | 生成组件 |
|-----------|--------|---------|
| `parent_id` | 树形父级 | TreeSelect（单选） |
| `*_id`（外键） | 外键关联 | 自动查关联表显示字段 |
| `*_ids` | 多选外键 | Select（多选） |
| `password` | 密码字段 | InputPassword，bcrypt 加密 |
| `*_url` / `*_link` | URL | Input（带 https:// 前缀） |
| `avatar` / `*_image` / `*_img` | 图片 | ImageUpload |
| `*_file` | 文件 | FileUpload |
| `content` / `*_text` / `description` | 长文本 | Textarea |
| `sort` / `order` / `*_num` | 数字 | InputNumber |
| `icon` | 图标 | IconPicker |
| `*_at`（非公共字段） | 时间 | DateTimePicker |

## 树形表支持

当表中包含 `parent_id` 字段时，自动生成：

- 后端：`/tree` 接口（递归组装树形结构）+ `/list` 接口
- 前端：`treeConfig` 树形表格（无分页，自动展开）
- 表单：TreeSelect 选择上级节点

## 关联字段显示

`*_id` 外键字段自动查找关联表的显示字段（优先级：title > name > username），在列表和详情中显示关联名称而非 ID。

## 生成后的手动工作

代码生成器生成的是基础 CRUD 代码，以下功能需要手动补充：

1. **路由注册**：将生成的 router 文件注册到 `cmd.go`
2. **特殊业务逻辑**：如授权、数据权限等
3. **特殊 UI 组件**：如授权弹窗、树形选择器等
4. **DAO 层**：运行 `gf gen dao` 生成（不由代码生成器负责）

## 铁律

> 如果生成的代码有问题，先修改代码生成器模板，再重新生成。不要手动修改生成的代码。
